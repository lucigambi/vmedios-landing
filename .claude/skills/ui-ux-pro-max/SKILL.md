## Layout Patterns

- Card Grid (Dashboard)
- Hero + Content Blocks
- Sidebar Navigation
- Top Navigation
- Sticky Header
- Footer with Sections

## Component Guidelines

**Buttons:**

- Minimum 44px height for touch targets
- Clear visual states: default, hover, active, disabled
- Icon + text or text only

**Forms:**

- Clear labels above inputs
- Visible focus states
- Error messages below field
- Placeholder text is not a label

**Navigation:**

- Clear hierarchy
- Active state indication
- Mobile-responsive
- Keyboard accessible

**Cards:**

- Consistent shadows/elevation
- Clear hierarchy within card
- Action buttons clear
- Responsive sizing

## Accessibility Requirements

- Color contrast WCAG AA minimum (4.5:1 text)
- Semantic HTML structure
- ARIA labels where needed
- Keyboard navigation support
- Focus indicators visible
- Alt text for images
- Clear link text

## Anti-Patterns to Avoid

- Generic SaaS card grids as primary layout
- Carousels without clear purpose
- Motion without meaning
- Contrast below WCAG AA
- Unlabeled form inputs
- Overuse of gradients
- Too many competing colors
- Inconsistent spacing

## When to Use This Skill

- Designing landing pages, dashboards, apps
- Creating UI components
- Choosing color schemes and typography
- Reviewing designs for consistency
- Making accessibility decisions
- Selecting style direction

## Your Process

1. Ask about the product type and industry
2. Suggest a design direction and color palette
3. Define typography system
4. Create layout structure
5. Build components with accessibility in mind
6. Review for consistency and WCAG compliance
